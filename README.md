## 3. Technologies Used

Git - Perfect Palate used Git as the version control system during its development.

GitPod - An online Integrated Development Environment (IDE).

HTML5 - HTML stands for Hypertext Markup Language and it is the backbone of Perfect Palate. The latest version (HTML5) - was used to add and structure the site content.

CSS3 - CSS stands for Cascading Style Sheets. The latest version (CSS3) was used to style all HTML content on the website for enhanced structure and visual appeal.

Materialize - Perfect Palate uses Materialize's grid system to layout site content in a structured manner. The framework also provides some assistance to styling with class names.

Google Fonts - This font library offers a wide variety of font styles to help with the Perfect Palate's brand. This project uses Oleo and Lato fonts.

Font Awesome - Perfect Palate uses this icon library with its great selection of icons, to improve user experience (UX).

JavaScript - onclick functionality to print full recipe or save as PDF.

JQuery - Perfect Palate uses the JQuery Framework in its script. It also assists Materialize functionalities by initializing them e.g. the loading of modal forms.

Python - General-purpose coding language used to render Perfect Palate's templates and run backend functionalities including wiring to database on MongoDB.

Flask - Micro-web framework written in Python. Perfect Palate uses this to write the python app functions and app routes.

Jinja - A web template engine. Perfect Palate uses this to unpack the requested data retrieved from MongoDB in the HTML templates.

Pytest - Testing framework used for writing test codes in python for refactored code in the Perfect Palate app.

Werkzeug - Used to generate and check password hashes for extra security as user data is submitted to the database from the Perfect Palate Login and Register pages.

MongoDB - A non-relational (NoSQL) source-available cross-platform document-oriented database program used to store data submitted from Perfect Palate. 

Balsamiq - Used to construct Perfect Palate's wireframes in the skeleton phase of UX.

Mindmup - Used to brainstorm ideas in the conceptual phase of Perfect Palate's project development.
